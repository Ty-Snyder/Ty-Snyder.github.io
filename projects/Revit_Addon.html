Revit Addon

Workflow Automation Addon

Project Overview

What began as a pyRevit intern project to make simple tools that evolved into a comprehensive, 25,000-line C# enterprise solution for Intech Mechanical I have worked on for the last 2 years.

After identifying limitations or speed concerns in existing Revit Addons, I rebuilt the toolset from the ground up to ensure stability, performance, and cross-version compatibility. The tool replaces expensive third-party plugins with custom tools built from direct industry insight.

Key Technical Achievements:

  • Custom Collision Engine: Developed a shape collision system using implicit surfaces and Principal Component Analysis (PCA) for rotation correction.
  • Multi-Version Architecture: Implemented a dependency network using abstract classes and multi-targeting to support various Revit API versions and .NET frameworks.
  • Enterprise Scalability: Transitioned the codebase to a structured C# environment to prevent "addon-clash" and direct integration with Revits core API.

Custom Collision Engine Architecture

Revit's native collision engine lacks support for specific element types and struggles with performance at scale. To solve this, I engineered a custom intersection algorithm tailored for placing sleeves at wall-pipe/duct intersections. This system balances the speed of box-box intersection with the accuracy of mesh-based methods, bypassing the need for heavy ray-casting algorithms like Möller–Trumbore.

01

Geometry Extraction & Decoupling

Revit's API prevents simultaneous calls, making multi-threading impossible with native geometry. My solution extracts a simplified point-and-vertex representation of the target elements immediately. By decoupling this data from the Revit API, I can process collisions on separate threads.

Optimization: Walls are simplified to double-sided planar meshes (excluding core layers) to prevent false positives on internal piping.

02

PCA Orientation (OBB)

Standard Axis-Aligned Bounding Boxes (AABB) create large "dead zones" around rotated pipes. I utilize Principal Component Analysis (PCA) to calculate the covariance matrix of the element's point cloud. This yields the primary eigenvectors, allowing the system to generate a tight-fitting Oriented Bounding Box (OBB) aligned with the object's natural rotation.

03

Spatial Partitioning

To avoid an O(n^2) complexity check, I implemented a 3D spatial partitioning system. The model is divided into a grid of partitions. Walls and pipes are registered to their respective grid cells. The intersection algorithm only runs on partitions containing both a wall and a pipe, instantly eliminating calculations for non-adjacent elements and drastically reducing CPU time.

04

Implicit Surface Calculation

If the Bounding Boxes overlap, the system verifies the collision using Implicit Surfaces (Signed Distance Functions). Instead of complex mesh intersection, I plug the coordinates into the implicit formula for a rotated prism/cylinder. A negative result confirms the point is inside the shape, while a positive result confirms it is outside. This mathematical approximation is significantly faster than ray-casting.

05

Intersection Verification

Since walls are rectangular, their implicit surfaces are highly accurate. I check the mesh points of the pipe/duct against the wall's implicit definition. To handle edge cases (e.g., a pipe passing through a wall without its vertices landing inside the wall volume), the system also runs a plane-vertex intersection check to ensure 100% collision accuracy.

06

Analysis & Placement

Once a collision is confirmed, the intersection points are projected onto a 2D plane aligned with the wall. I apply a Least Squares regression to fit both a circle and a rectangle to the points, comparing the R^2 values to automatically determine if a round or rectangular sleeve is required.

Benchmark: The algorithm places ~200 sleeves in 12 minutes (placing 1 sleeve every ~3.6 seconds). Competitor software took 1 hour+ for the same task and lacked logic for duct work or fitting.
